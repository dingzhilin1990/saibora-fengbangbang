## 三、评审标准

评审采用**四维评分制**，每个维度 1-10 分：

| 维度 | 权重 | 说明 |
|------|------|------|
| 技术突破（Technical Breakthrough） | 40% | 架构创新、性能提升、能力边界拓展 |
| 生态影响（Ecosystem Impact） | 30% | 用户规模、开发者采用、第三方集成 |
| 开源贡献（Open Source Contribution） | 20% | 代码开源、权重开放、研究开放 |
| 社会价值（Social Value） | 10% | 安全性、可解释性、正面社会影响 |

**综合得分** = (技术×0.4 + 生态×0.3 + 开源×0.2 + 社会×0.1)

## 五、封神流程

```
提名(Nomination) → 初审(Initial Review) → 候选(Candidate) → 终审(Final Review) → 封神(Deification)
```

1. **提名**：任何人可提名任何 AI 模型/产品
2. **初审**：由维护者进行初步审核，确保信息真实
3. **候选**：通过初审后进入候选榜，等待完整评审
4. **终审**：评审委员会按四维框架打分
5. **封神**：正式赋予神号，移入正式封神榜

## 六、封神委员会

> 封神委员会是赛博封神榜的最高评审机构。

**当前委员会成员**（开放申请）：
- 项目维护者（创始委员会）
- 贡献者代表（按 PR 活跃度邀请）

**成为委员**：
- 为本项目做出重大贡献（整理数据、提交评审报告等）
- 向 `committee@saibora.example.com` 申请

## 七、封神规则

1. **一人一神号，不可重复**
2. **封神须有公论，禁止私下授受**
3. **劫神同样需要正式评审，不可随意定性**
4. **所有封神记录永久保存，不可抹去**
5. **对封神结果有异议，可提交申诉**

## 八、注意事项

- 请确保提名信息的**准确性**，禁止凭空捏造

- 劫神提名需要特别谨慎，需要提供充分证据
- 请尊重不同文化背景和政治立场
